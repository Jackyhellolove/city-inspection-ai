# 城市巡查AI助手

## 项目介绍

利用无人机、人工智能技术，
辅助城市设施巡查和运维管理。

## 主要功能

- 无人机巡查数据管理
- AI识别城市问题
- 自动生成巡查报告
- 问题闭环管理

## 应用场景

- 城市道路
- 园林环卫
- 市政设施
- 城市生命线

完成项目框架

第二阶段：
接入AI识别

第三阶段：
开发管理平台

## 运行方法

### 桌面界面

在项目文件夹的终端中运行：

```bash
python3 app.py
```

输入巡查问题并点击“开始识别”，即可查看问题类型、处理部门和优先级。

如果 Mac 上的桌面控件显示不正常，请使用更稳定的浏览器版：

```bash
python3 web_app.py
```

启动后，终端会同时显示 Mac 地址和同一 Wi-Fi 下的手机访问地址。将手机访问地址输入手机浏览器即可预览；如 macOS 弹出网络访问提示，请选择“允许”。

浏览器版支持自动生成问题编号，记录发现地点、负责人和处置状态，并可保存历史记录和导出 CSV 表格。记录仅保存在当前浏览器中。

台账页面还支持状态统计、关键词搜索、按处置状态筛选和删除单条记录。

已保存的巡查记录可以继续编辑问题描述、发现地点、负责人和处置状态，系统会记录最后更新时间并实时刷新统计结果。

为防止浏览器数据被清理，页面支持将全部台账备份为 JSON 文件、从备份恢复新记录，以及直接打印巡查报告。

数据看板会实时展示问题总数、各处置状态数量、办结率、办结进度和问题类型分布。

每条巡查问题可设置完成期限，系统会自动显示即将到期和已逾期预警，并支持按期限状态筛选；已完成的问题不再计入逾期统计。

巡查记录支持添加一张现场照片：手机可直接调用相机拍摄，电脑可选择图片。照片会在保存前自动压缩，台账中显示缩略图并可点击查看大图，完整数据备份会同时包含照片。

每次修改处置状态时可以填写处理说明，系统会自动记录更新时间、原状态和新状态；点击台账中的“详情”按钮，可以查看问题完整信息和处置时间线。

台账支持勾选当前筛选结果中的多条记录，一次性指定负责人或修改处置状态。批量操作同样会写入每条问题的处置时间线。

单条记录使用正式编辑窗口集中修改问题描述、地点、负责人、状态、完成期限和处理说明，不再需要连续操作多个浏览器提示框。

新增记录时可以获取设备当前位置并保存经纬度，台账和详情窗口会提供地图查看入口；CSV 导出同时包含坐标信息。局域网 HTTP 预览受手机浏览器安全策略影响时，可继续填写文字地点，正式 HTTPS 版本可使用自动定位。

巡查报告支持填写报告名称、单位和编制人，自动使用当前筛选结果生成报告周期、统计摘要和签字栏，可通过浏览器打印或保存为 PDF。

项目已支持 PWA 安装和基础离线缓存：部署到 HTTPS 后，可在手机浏览器中添加到主屏幕；支持安装提示的浏览器会显示“安装应用”按钮。iPhone 可在 Safari 的分享菜单中选择“添加到主屏幕”。

当前版本已接入 Supabase 真实登录能力。请将 `supabase-config.example.js` 复制为 `supabase-config.js`，再填写项目 Settings > API 中的 Project URL 和 publishable key。该配置文件不会提交到 GitHub；不要填写 secret key 或 service_role key。

巡查记录云端同步启用后，新增、编辑、批量修改和删除会同步到 Supabase 的 `inspection_records` 表；首次登录会读取当前账号有权限查看的云端台账。要让已打开的手机和 Mac 页面自动刷新，请在 Supabase SQL Editor 中运行 `supabase-realtime.sql`。

运行 `supabase-timeline.sql` 后，创建记录、编辑记录和批量处理产生的状态变化及处理说明会写入 `inspection_updates` 表；问题详情将从云端读取并展示完整处置时间线。

运行 `supabase-photos.sql` 后，新增巡查记录时选择的现场照片会压缩后上传至私有的 Supabase Storage 桶。巡查员只能查看自己上传的照片，管理员可查看全部照片；换设备登录后照片仍可显示。

巡查编号采用日期加随机标识生成，避免不同设备同时录入时发生重复。云端短暂不可用时，记录会暂存本机，并在下次打开页面后自动重试同步。

运行 `supabase-attachments.sql` 后，可在编辑记录时一次选择最多 6 张整改凭证照片；每条记录可分多次持续追加，详情页按时间展示照片、处理说明和上传时间。
